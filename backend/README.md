# üöÄ Hackathon Backend - AI Itinerary Generator API

Django REST Framework backend for an AI-powered itinerary generation app.

## üìã Features

- ‚úÖ Token-based authentication (no cookies, works with separate frontend)
- ‚úÖ User registration and login
- ‚úÖ CRUD operations for itineraries
- ‚úÖ CRUD operations for itinerary items
- ‚úÖ **Smart append & insert methods** (automatic order management)
- ‚úÖ **Move & delete with auto-reordering** (no gaps in order)
- ‚úÖ ML-powered itinerary generation endpoint (ready for your model)
- ‚úÖ CORS configured for cross-origin requests
- ‚úÖ Order-based itinerary items (replaces linked list with efficient querying)

> üí° **NEW!** Easy-to-use methods for appending and inserting items. See `APPEND_INSERT_SUMMARY.md` for details!

## üóÑÔ∏è Database Model Explanation

### Why We Don't Use a Linked List

Your original idea was to use a linked list structure:
```
Item(id: 1, data: "Go to CN Tower", next_item_id: 3)
Item(id: 2, data: "Get breakfast", next_item_id: 1)
```

**Problem:** This requires **N separate database queries** to get the full itinerary (N+1 query problem). Very slow!

**Solution:** We use an `order` field instead:
```
Item(id: 1, data: "Go to CN Tower", itinerary_id: 1, order: 1)
Item(id: 2, data: "Get breakfast", itinerary_id: 1, order: 0)
```

**Benefits:**
- ‚úÖ Get entire itinerary in 1 query: `SELECT * FROM ItineraryItem WHERE itinerary_id = 1 ORDER BY order;`
- ‚úÖ Reordering is trivial (just update the `order` field)
- ‚úÖ Database automatically sorts items

## üöÄ Deployment Tips

### For Hackathon Demo (Quick & Dirty)

Keep using SQLite and `python manage.py runserver`. Just make sure CORS is enabled (it already is).

### For Production (Post-Hackathon)

1. **Change `SECRET_KEY`** in `settings.py`
2. **Set `DEBUG = False`**
3. **Update `ALLOWED_HOSTS`** with your domain
4. **Use PostgreSQL** instead of SQLite
5. **Restrict CORS** to your frontend domain:
   ```python
   CORS_ALLOWED_ORIGINS = [
       "https://your-frontend.com",
   ]
   ```
6. **Use Gunicorn + Nginx** instead of `runserver`
7. **Deploy to Heroku/DigitalOcean/AWS**

## üêõ Troubleshooting

### CORS Error: "No 'Access-Control-Allow-Origin' header"

‚úÖ Already fixed! `CORS_ORIGIN_ALLOW_ALL = True` is set in `settings.py`.

### "Invalid token" error

Make sure you're sending the token in the correct format:

(Not `Bearer abc123...`)

### Can't login with admin panel

Run `python manage.py createsuperuser` first.

Run migrations:
